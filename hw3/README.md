
# Dessert Planner (HW3)

這是一個純前端應用程式，使用 React + TypeScript 開發，透過 CSV 資料檔案來展示甜點與咖啡商品。應用程式提供完整的購物車功能與訂單管理系統。

## 🍰 服務功能

### 主要功能
1. **商品瀏覽與篩選**
   - 瀏覽所有甜點與咖啡商品
   - 依類別篩選（糕點、蛋糕、咖啡、馬卡龍、塔類、甜點）
   - 關鍵字搜尋功能
   - 價格排序（低到高、高到低、評分）

2. **購物車系統 ("今日計畫")**
   - 將喜愛的商品加入購物車
   - 調整商品數量
   - 移除不需要的商品
   - 即時計算總金額
   - 資料自動儲存至本地儲存空間

3. **訂單提交與管理**
   - 提交購物車內容為正式訂單
   - 可添加備註說明
   - 查看歷史訂單記錄
   - 訂單狀態管理（已提交、要求修改）

### 特色功能
- **響應式設計**：支援各種螢幕尺寸
- **無障礙設計**：包含跳過連結等無障礙功能
- **視覺效果**：使用粒子動畫與玻璃質感設計
- **本地儲存**：購物車與訂單資料自動保存
- **即時更新**：修改 CSV 檔案後自動熱重載

## 🚀 使用方式

### 1. 瀏覽商品
- 進入首頁後，可以看到所有可用的甜點與咖啡商品
- 使用頂部的篩選列來：
  - 搜尋特定商品名稱
  - 選擇特定類別
  - 排序商品（價格、評分）

### 2. 建立今日計畫
- 點擊商品卡片上的「加入計畫」按鈕
- 點擊右上角的「Today's Plan」按鈕開啟購物車
- 在購物車中可以：
  - 調整商品數量（+/- 按鈕）
  - 移除不需要的商品（X 按鈕）
  - 查看總金額

### 3. 提交訂單
- 在購物車中點擊「提交計畫」按鈕
- 可以添加備註說明（選填）
- 點擊「Submit」完成訂單提交
- 訂單會自動儲存並可在「Orders」頁面查看

### 4. 管理訂單
- 點擊導航列的「Orders」查看所有歷史訂單
- 每個訂單顯示：
  - 訂單 ID 與建立時間
  - 商品清單與數量
  - 總金額
  - 備註內容
  - 訂單狀態

## 📁 資料檔案結構

將 CSV 檔案放置在專案的以下位置：

```
/public/data/items.csv
/public/data/categories.csv
```

### items.csv 格式
包含商品詳細資訊：
- `id`: 商品唯一識別碼
- `name`: 商品名稱
- `category`: 商品類別
- `price`: 價格
- `description`: 商品描述
- `image_url`: 商品圖片 URL
- `tags`: 標籤（逗號分隔）
- `rating`: 評分
- `stock`: 庫存數量
- `location`: 地點
- `opening_hours`: 營業時間

### categories.csv 格式
定義商品類別：
- `id`: 類別識別碼
- `display_name`: 顯示名稱
- `icon`: 圖示名稱

## 🛠 技術堆疊

- **前端框架**: React 18 + TypeScript
- **建置工具**: Vite
- **樣式框架**: TailwindCSS
- **UI 元件庫**: shadcn/ui + Radix UI
- **資料處理**: PapaParse (CSV 解析)
- **動畫效果**: Framer Motion
- **路由管理**: React Router DOM
- **狀態管理**: React Context API

## 🔧 本地開發設定

### 環境需求
- Node.js 16+ 
- npm 或 yarn

### 安裝與執行

```bash
# 1. 複製專案
git clone <repository-url>
cd hw3

# 2. 安裝相依套件
npm install

# 3. 確認 CSV 檔案位置
# 將 items.csv 和 categories.csv 放在 /public/data/ 目錄下

# 4. 啟動開發伺服器
npm run dev

# 5. 開啟瀏覽器
# 預設會開啟 http://localhost:5173
```

### 其他指令

```bash
# 建置生產版本
npm run build

# 預覽生產版本
npm run preview

# 類型檢查
npx tsc --noEmit
```
